Theme Architecture Approaches

How should the theme handle header, footer, menus, and sidebars?

A

Full Block-Based (Gutenberg Native)

Header/Footer/Sidebar are WordPress template parts using Gutenberg blocks. Menus are block-ized. Agents work via block JSON/REST API.

B

JSON Config + Minimal Templates

Header/Footer/Menu structure defined in JSON config files. Templates are dumb PHP shells. Agents edit JSON directly; humans use a settings UI.

C

Decoupled / Headless-Ready

Theme is a clean HTML/JS layer over WordPress as a headless CMS. Agents talk to WP REST/GraphQL, render via static files or CDN. Max agent compatibility.

Key Tradeoffs

  • A: Native feel, but Gutenberg still fights agents
  • B: Simple, predictable, agent-friendly JSON
  • C: Maximum agent compatibility, harder human editing
